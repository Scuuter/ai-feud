# **Game Design Document (GDD) & Technical Specification**

**Project Name:** Survey Says: AI Feud

**Genre:** Web-based Casual / Trivia

**Target Vibe:** Minimalist, quirky, fast-paced (e.g., neal.fun, Wordle, Infinite Craft)

## **1. Executive Summary**

A web-based party/trivia game blending the mechanics of *Family Feud* with the unpredictable, hallucinatory nature of AI. Players attempt to guess the top answers to absurd topics, generated by surveying 100 AI personas. The core hook lies in the juxtaposition of rigid trivia mechanics and unhinged, creative AI responses.

## **2. Product Decisions & Phases overview**

### **Phase 1: MVP (Minimum Viable Product)**

**Goal:** Prove the core gameplay loop is fun and achieve zero-friction onboarding.

* **Mode:** Single-player (Solo).  
* **Content:** 50 pre-generated, highly curated, absurd topics, 100 different generated AI personas answering these questions  
* **Onboarding:** No login, no accounts. Instant play upon page load.  
* **Core Loop:**  
  1. Read Question (e.g., "Name a weapon a wizard would use if they lost their wand").  
  2. Type guess into a minimalist text input.  
  3. Receive feedback: Reveal board tile (correct) or get a Strike (incorrect).  
  4. Win (clear board) or Lose (3 strikes).  
* **Viral Loop:** "Share your score" button copying a Wordle-style emoji grid to the clipboard.  
* **Validation:** Strict exact matching and fuzzy matching (typo tolerance) against static synonyms. No external API calls.

### **Phase 2: Multiplayer & Dynamic Generation (V1)**

**Goal:** Introduce social play and endless replayability.

* **Mode:** Real-time Multiplayer Lobby (Host + Players sharing a 4-letter room code).  
* **Dynamic Generation:** Host selects a "Demographic" (e.g., 19th Century British Aristocrats) and a "Theme".  
* **Loading UX:** The "Pre-Game Show" — entertaining, real-time UI updates while the backend orchestrates the AI generation and clustering.  
* **AI Validation:** Uses a lightweight Edge function to validate user input via semantic similarity if exact/fuzzy matching fails.
* **Gameplay:** Real-time buzzer or turn-based text input.
* **Data Automation:** Automated CI/CD pipeline via GitHub Actions to validate and seed MongoDB with new data.

### **Phase 3: Mobile Expansion (V2)**

**Goal:** App Store presence.

* **Platform:** iOS and Android via React Native.  
* **Enabler:** Strictly decoupled business logic from UI components in Phase 1 & 2.

## **3. Technology Stack Choice & Justification**

*Focus: Widespread, native, highly maintainable, and portable to mobile.*

* **Frontend:** **React + Next.js (App Router)**  
  * *Why:* Industry standard, massive ecosystem, easy deployment. Using React ensures that state management and business logic can be lifted and shifted to React Native later.  
* **Backend:** **Node.js (TypeScript) + Serverless Edge Functions**  
  * *Why:* Keeps the stack unified. Edge functions allow for low-latency AI validation close to the user.  
* **Database:** **MongoDB (NoSQL)**  
  * *Why:* The game data is naturally document-oriented. AI-generated responses can be stored as nested JSON objects (Question -> Clustered Answers -> Synonyms) without the overhead of relational mapping or complex joins.  
* **Styling & Animation:** **Tailwind CSS + Framer Motion**  
  * *Why:* Tailwind allows for rapid, consistent, minimalist UI design. Framer Motion handles the "card flip" and "strike" animations crucial for the "neal.fun" feel.  
* **Real-time Engine (V1):** **PartyKit or Supabase Realtime**  
  * *Why:* Serverless WebSockets remove the need to manage a persistent Socket.io server. These tools handle scaling and lobby state management automatically.  
* **AI Engine:** **Local LLM for data generation / Cheapest model for answer validation**  
  * *Why:* Cost-effective, fast, and excellent at adhering to JSON schemas for data clustering.

## **4. Required Technical Components**

### **Layer A: The Offline Data Pipeline (For MVP Data Prep)**
location: scripts/data-generation
*These are Node.js scripts run locally to populate the MVP database.*

1. **The Surveyor (survey.ts):** Takes a list of topics and a demographic, asking the LLM to output an array of 100 short, punchy responses in a single API call.  
2. **The Cluster Engine (cluster.ts):** A multi-stage map-reduce script that extracts 5-8 core concepts using the original topic context, maps the 100 responses to these clusters or wildcards, and normalizes the cluster scores to sum to 100.
3. **The Enrichment Engine (enrichment.ts):** A final script that passes over the normalized clusters to generate 3-5 game-show synonyms per answer, and selects a unique persona per cluster/wildcard to generate a rich flavor quote. Output is saved as a single JSON document per question.

### **Layer B: The Backend API (Game Server)**
location: src/lib/game-logic
1. **Game State Manager (GameState.ts):**  
   * Manages round state, strikes, and scores.  
   * Logic is pure TypeScript to ensure portability between Web, Backend, and Mobile.  
2. **Semantic Matcher (Matcher.ts):**  
   * **Step 1:** Exact/Lowercase match.  
   * **Step 2:** Fuzzy matching (Levenshtein) for typos.  
   * **Step 3:** Semantic AI validation (fast LLM call) to check conceptual similarity.  
3. **Document Schema (NoSQL):**  
See the [schema.md](schema.md) file.

### **Layer C: The Frontend Architecture (React)**
location: src/app, src/components

1. **useGameLoop Hook:** Custom hook managing the local state.  
2. **<Board /> and <Tile /> Components:** The visual grid using Framer Motion for 3D flip effects.  
3. **<InputTerminal /> Component:** Focused minimalist input that clears on Enter.  
4. **<StrikeIndicator /> Component:** Large visual feedback for incorrect guesses.

## **5. Development Milestones**

* **Milestone 1: The Engine Room.** Build the survey.ts and cluster.ts scripts. Generate the first 20 absurd MVP questions.  
* **Milestone 2: The Core UI.** Build the React frontend. Perfect the tile flip animations and input feel with static JSON data.  
* **Milestone 3: Semantic Matcher.** Implement the multi-step validation logic and connect the frontend to the MongoDB instance.  
* **Milestone 4: Multiplayer & Launch.** Integrate PartyKit for lobby management and deploy to Vercel.
